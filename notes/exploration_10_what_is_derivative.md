## 1. 미분에 대한 3단계 오해 → 깨달음 (브로의 학습 궤적)

### 📚 1단계: 학교 시절 — "미분 = 공식 외우기"

```
x² → 2x
x³ → 3x²
sin(x) → cos(x)
```

교과서는 이렇게 가르친다. 미분 = **"도함수를 구하는 공식적 조작"**. "순간 변화율"이란 말은 들었지만, 공식 적용이 본질이라고 믿게 됨. 수포자에겐 그냥 **외워야 하는 수학적 의식**.

→ **문제**: 공식이 "왜" 필요한지, 어디에 "쓰이는지" 안 알려줌.

> 💡 **수포자 자책하지 말 것**: 학교 수학은 "도구의 용도"를 안 가르치고 "조작법"만 반복하게 하는 구조적 한계가 있음. 브로가 몰랐던 건 브로 잘못이 아니라, **제대로 된 응용 맥락이 없었던 것**. 이제 그 맥락(딥러닝)을 만났으니 이해가 시작되는 것.
>
> 📖 **참고 — Paul Lockhart, "A Mathematician's Lament" (2002)**:
> 수학자 폴 로크하트가 쓴 유명한 수학 교육 비판 에세이. "학교 수학은 음악을 작곡하는 법은 안 가르치고 악보 읽는 법과 음계 외우기만 반복하는 것과 같다"고 비유. "규칙과 공식을 기계적으로 외우게 하면서, 수학이 본래 창조적 탐구 활동이라는 사실은 숨긴다"는 게 핵심. 브로가 겪은 "공식만 외우고 용도는 몰랐다"는 경험은, 로크하트가 비판한 시스템의 정확한 산물. → [원문 PDF](https://worrydream.com/refs/Lockhart_2002_-_A_Mathematician's_Lament.pdf), [Fermat's Library 주해판](https://fermatslibrary.com/s/a-mathematicians-lament)
>
> 🤖 **[가짜 인용에서 건진 통찰 — AI 환각 기록]**
> 처음엔 "수학자 제이슨 윌콕스의 말: '학생이 미분을 이해하는 순간은 공식을 외울 때가 아니라, 직접 예측에 썼을 때다'" 라는 인용을 제안받았으나, **이 인물은 AI가 생성한 가짜**였음 (실존하지 않음). 하지만 **통찰 자체는 진짜** — "이해는 응용/경험을 통해 도달한다"는 학습 과학의 **구성주의(constructivism)** 이론과 일치. 가짜 출처를 빼고 본질만 남김. (교훈: AI가 인용을 줄 땐 출처 검증 필수. 단, 통찰의 가치는 출처와 별개일 수 있음)

### 🎓 2단계: 교양서/강의 — "미분 = 예측 도구"

"미분은 뭔가를 예측하는 데 쓰인다" — 교양 수학 책에선 이렇게 설명. 뉴턴이 행성 운동을 예측하고, 경제학이 변화율을 예측하는 데 미분을 쓴다고.

이론적으론 납득되나, **"내가 직접 예측 문제를 풀어본 경험"이 없으니** 마음에 와닿지 않음. 여전히 추상적.

→ **문제**: "예측한다"는데 **뭘 어떻게 예측한다는 건지** 감이 안 옴.

### 🔥 3단계: 딥러닝 학습 — "미분 = 실제로 쓰이는구나!" (아하 모먼트)

`3권 step04 수치 미분`을 보는 순간:

> "잠깐, **코드로 짠 함수를 그대로 미분**한다고? `f(x+h) - f(x-h)`를 계산하면 기울기가 나온다고? 그럼... **if문 있는 함수도? for문 있는 함수도? 심지어 신경망도?**"

그리고 뒤이어:

> "아, **신경망 학습이 그래디언트(=미분값)로 파라미터를 조정**하는 거잖아. 이게 바로 미분이 **실제로 쓰이는 모습**이었구나!"

→ **깨달음**: 미분은 단순히 도함수 구하는 게 아니라, **"어느 방향으로 움직이면 더 나아질까?"**를 알려주는 도구. 신경망 학습이 그 도구의 가장 강력한 응용.

## 2. 미분의 본질: "순간 변화율"이란 무엇인가?

### 정의

> **미분(도함수) `f'(x)`** = 함수 `f`가 `x` 근처에서 **얼마나 빠르게 변하는지**를 나타내는 값. 즉 **"순간 변화율(instantaneous rate of change)"**.

### 직관 — "조금 움직였을 때 얼마나 변하나?"

```
f(x) = x²

x=2에서:
  f(2) = 4
  f(2.001) ≈ 4.004    → 0.001 움직였을 때 0.004 변화
                       → 변화율 ≈ 0.004 / 0.001 = 4
                       → f'(2) = 4 (= 2x에서 x=2 대입)
```

→ 미분값이 4라는 건: "x=2에서 x를 1만큼 움직이면, f는 약 4만큼 변한다"는 의미. 즉 **그 지점에서의 기울기**.

### "순간"이라는 말의 함정

학교에선 "순간(instantaneous)"을 강조하지만, 이게 오히려 헷갈림을 유발. 진짜 의미는:

> "x를 **무한히 작게** 움직였을 때의 변화율의 극한"

하지만 실용적으론:

> "x를 **아주 작게** 움직였을 때의 변화율의 근사치"

→ 이 "아주 작게"를 코드로 구현한 게 **수치 미분** (step04). 무한소(limit) 대신 `eps=1e-4` 같은 작은 값을 사용.

## 3. ★ 코드로 짠 함수도 미분된다 (autograd의 기반)

이게 브로의 **핵심 아하 모먼트**.

### 전통적 미분의 전제 (해석적)

전통적 미분은 함수를 **수식**으로 다룸:
- `f(x) = x²` → 공식 적용 → `f'(x) = 2x`
- 사람이 **함수의 형태를 알고 있어야** 미분 가능

### 수치 미분의 파괴력

수치 미분은 함수를 **블랙박스**로 다룸:
- `f`가 뭔지 모름. 그냥 "어떤 입력 → 어떤 출력"을 주는 것만 앎
- `f(x+h)`를 한 번 불러서 출력을 얻으면 됨

```python
def numerical_diff(f, x, eps=1e-4):
    x0 = Variable(x.data - eps)
    x1 = Variable(x.data + eps)
    y0 = f(x0)      # ← f를 "호출"만 하면 됨. 내부 구조 몰라도 됨
    y1 = f(x1)
    return (y1.data - y0.data) / (2 * eps)
```

### ★ 그럼 어떤 함수든 미분 가능?

브로의 질문 (Q3): "관찰 가능한 대상이라면 전부 다 가능은 할 것 같은데?"

**대체로 맞음** — 단, 몇 가지 조건:

| 함수 종류 | 수치 미분 가능? | 비고 |
|---|---|---|
| `f(x) = x²` (수식) | ✅ | 전통적 케이스 |
| `f(x) = x² + if x > 0: x*2` (if문) | ✅ | if문 있어도, 입력→출력만 관측 가능하면 OK |
| `f(x) = sum(for i in range(int(x)))` (for문) | ✅ | 마찬가지 |
| `f(x) = neural_network(x)` (신경망) | ✅ | ★ 핵심: 코드로 짠 함수도 미분 가능 |
| `f(x) = random()` (랜덤) | ❌ | 매번 다른 출력 → 미분값이 의미 없음 (브로 직관!) |
| `f(x) = |x|` (절댓값, x=0) | ⚠️ | 수학적으론 미분 불가능 (꼭짓점). 수치적으론 0 근처에서 불안정 |

→ 브로가 "일관성 없는 출력이면 의미 없다"고 한 게 정확함. **미분이 의미 있으려면 함수가 "결정적(deterministic)"이고 "연속적(continuous)"이어야**.

### 왜 이게 혁명적인가?

**신경망은 수학 공식이 아니라 "코드"**야. 수백만 개의 if문(ReLU), 행렬 곱셈, 활성화 함수가 얽힌 거대한 프로그램. 전통적 미분으론 손으로 공식 유도 불가능.

그런데 수치 미분(그리고 더 정확한 역전파)을 쓰면, **코드로 짠 신경망도 미분 가능**. 이게 딥러닝이 "학습"할 수 있는 수학적 기반.

> 💡 **autograd의 철학**: 함수의 내부를 몰라도, 입력과 출력만 관측할 수 있으면 미분할 수 있다. PyTorch, TensorFlow, DeZero 모두 이 철학.

## 4. 미분값을 구하는 두 가지 방법: 해석적 vs 수치적

### 해석적 미분 (analytic differentiation)

사람이 **수학 공식**을 유도해서 적용:
- `f(x) = x²` → `f'(x) = 2x` (공식)
- `f(x) = sin(x)` → `f'(x) = cos(x)` (공식)
- 합성: `f(g(x))` → `f'(g(x)) · g'(x)` (chain rule)

✅ **장점**: 빠르고 정확 (오차 없음)
❌ **단점**: 사람이 공식을 알아야. 복잡한 함수(신경망)는 손으로 불가능

### 수치 미분 (numerical differentiation)

컴퓨터가 **정의**를 그대로 코드로:
- `f'(x) ≈ (f(x+h) - f(x-h)) / 2h` (중앙 차분)

✅ **장점**: 범용 (어떤 함수든, 코드든). 사람이 수학 몰라도 됨
❌ **단점**: 느리고 오차 있음 (부동소수점 한계)

### 비교 표

| 항목 | 해석적 | 수치적 |
|---|---|---|
| **방법** | 공식 유도 | 정의를 코드로 |
| **속도** | 빠름 | 느림 (함수를 여러 번 호출) |
| **정확도** | 완벽 | 근사 (오차 있음) |
| **범용성** | 낮음 (공식 알아야) | 높음 (어떤 함수든) |
| **신경망 적합성** | 수동 불가능 | 가능하지만 너무 느림 |

→ **따라서 딥러닝에 제3의 방법 필요**: "해석적이되 자동으로" = **역전파**. step07+에서 등장.

## 5. 미분 불가능 / 의미 없는 미분 (Q3 심화)

브로가 "미분 불가능의 대상이 있는 것은 아닌가?" 질문. 수학적으로 정리.

### 수학적 미분 불가능점

함수가 **특정 점에서 미분 불가능(not differentiable)** 인 경우:

#### 케이스 1: 꼭짓점 (corner)

```python
def f(x):
    return abs(x)        # |x|
# x=0에서: 왼쪽 기울기=-1, 오른쪽 기울기=+1 → "기울기"가 안 정의됨
# 수치 미분해 보면 (f(0+h)-f(0-h))/(2h) = 0이 나오지만, 진짜 미분값은 없음
```

#### 케이스 2: 불연속 (discontinuity)

```python
def step(x):
    return 1 if x > 0 else 0   # 계단 함수
# x=0 근처에서 갑자기 0→1로 점프. "순간 변화율"이 무한대.
```

#### 케이스 3: 진동 (oscillation)

```python
def f(x):
    return sin(1/x) if x != 0 else 0   # x→0에서 무한 진동
# x=0 근처에서 기울기가 정의 불가
```

### 의미 없는 미분 (브로가 직관)

함수가 **결정적이지 않거나(랜덤)**, **관계가 없거나(노이즈)** 인 경우:

```python
import random
def f(x):
    return random.random()    # 입력 x와 무관한 랜덤 출력
# 수치 미분技术上으로는 계산할 수 있지만, 결과가 매번 달라서 의미 없음
```

→ 브로가 "측정해봐야 아무 의미 없어 보인다"고 한 것이 정확함.

### ★ 딥러닝과의 연결

신경망을 설계할 때 **미분 가능한 활성화 함수**만 써야 하는 이유:
- ✅ ReLU `max(0, x)`: 거의 다 미분 가능 (x=0만 예외, 무시 가능)
- ✅ Sigmoid `1/(1+e^-x)`: 매끄럽게 미분 가능
- ❌ Step function `1 if x>0 else 0`: 미분 불가능 → 학습 안 됨 (역사적으로 이 문제로 신경망이 발전 안 하다가, ReLU/sigmoid로 해결)

<a id="역전파"></a>
## 6. 왜 딥러닝에선 "역전파"가 필요한가?

수치 미분이 범용이고 코드에도 적용 가능하다면, **왜 굳이 역전파를?**

### 수치 미분의 치명적 한계: 느림

신경망의 파라미터가 1,000,000개라면:
- 수치 미분: 각 파라미터마다 2번 forward → **2,000,000번 계산**
- 역전파: forward 1번 + backward 1번 → **2번 계산**

→ 백만 배 차이. 수치 미분으론 실용적 학습 불가능.

### 역전파(backpropagation)의 아이디어

"chain rule을 자동으로, 그리고 효율적으로" 적용:
- forward: 입력 → 출력까지 계산하면서 중간값들 저장
- backward: 출력에서 입력 방향으로 chain rule을 한 번에 전파

결과: 해석적 미분의 **정확도** + 수치 미분의 **자동성**. 사람이 공식 몰라도 컴퓨터가 알아서 미분. ★

### 비교 — 세 가지 방법 정리

| 방법 | 정확도 | 속도 | 자동성 | 단계 |
|---|---|---|---|---|
| 해석적 (손계산) | 완벽 | 빠름 | ❌ (사람이) | 학교 |
| 수치 미분 | 근사 | 느림 | ✅ | step04 |
| 역전파 | 완벽 | 빠름 | ✅ | step07+ |

→ DeZero/PyTorch가 역전파를 택한 이유 명확.

> 🔬 **심화**: "왜 신경망은 역전파(리버스 모드)인가? 포워드 모드와 뭐가 다른가?" → [exploration_11_autodiff_modes.md](./exploration_11_autodiff_modes.md) — 자동 미분의 두 모드, 수치 미분의 캐싱 한계, 브로의 "포워드 방식 같다" 직관 검증

<a id="dezero여정"></a>
## 7. DeZero의 여정: 수치 → 역전파 → 고차 미분

이 책(3권)이 그리는 로드맵:

```
step04: 수치 미분 (지금)
   ↓ "범용이지만 느리고 오차 있음. 더 나은 방법은?"
step07+: 역전파 (해석적 자동)
   ↓ "신경망 학습 가능! 그런데 미분의 미분(2차)은?"
step25+: 고차 미분 (역전파를 다시 역전파)
   ↓ "뉴턴 방법 등 2차 최적화 기법까지"
```

**step04의 의미**: 수치 미분은 **"이정표"**. 진짜 목적지(역전파)에 도달하기 위해, "왜 더 나은 방법이 필요한지" 체감하게 해주는 다리.

## 8. 브로의 학습 메타인지 (이 탐구의 진짜 가치)

### 브로가 겪은 "미분의 와닿지 않음" — 왜 발생했나?

| 시기 | 미분 이미지 | 문제 |
|---|---|---|
| 학교 | "공식 외우기" | 본질(변화율) 안 알려줌 |
| 교양 | "예측 도구" (추상) | 직접 경험 없음 |
| 3권 | "실제 코드에 적용" ★ | 드디어 체감 |

→ 학교 수학의 근본적 한계: **"도구의 용도"를 안 가르치고 "조작법"만 가르침**.

### 아하 모먼트의 본질

이 탐구의 진짜 가치는 "미분 정의를 안다"가 아니라:
- **"내가 왜 이걸 몰랐지?"** 를 깨닫는 메타인지
- **"학교가 이렇게 가르쳤어야 했는데"** 를 체감
- **"이제 미분이 실용 도구로 보인다"** 는 인식 전환

<a id="일반학습법"></a>
## 9. ★ 일반 학습법 메타인지 (이번 탐구에서 파생)

> §8에서 다룬 건 "미분에 대한 메타인지"였다면, 이 섹션은 **더 일반적인 학습법에 대한 통찰**.
> 미분을 배우다 우연히 깨달은 것이지만, 모든 학습에 적용 가능한 원리들.

### 🎬 "이해했다고 착각하는 것" — 영상 시청의 함정

> 브로 고백: *"그랜트 샌더슨(3Blue1Brown)의 '에센스 오브 캘귤러스' 영상 시리즈를 몇 번 봤었어. 영상을 볼 때는 참 재미있고, 당시 이해한다고 생각만 했는데... 아, 착각만 했었던 것이지."*

이게 진짜 흔한 학습의 함정. **3Blue1Brown 영상은 "이해했다!" 착각을 일으키는 데 최적화**되어 있음. 이유:

1. **영상은 "보여주는 사람"이 모든 고민을 대신함** — 그랜트가 "이건 이렇게 생각하면 돼" 하고 답을 줌. 시청자는 "오 그렇네!" 하지만 **스스로 도달한 게 아님**
2. **"따라가는 것" ≠ "생각하는 것"** — 영상을 따라가는 건 수동적 추적(tracking). 진짜 이해는 스스로 질문을 만들고 답하는 것
3. **"오, 그렇구나!" ≠ "그래서 이걸 어디에 쓰지?"**** — 개념은 이해된 듯 but, 용도/응용을 안 물어봄. 그래서 며칠 지나면 까먹음

### 🔥 step04는 왜 달랐나?

그랜트 영상 vs step04 코드의 **결정적 차이**:

| | 3Blue1Brown 영상 | step04 코드 |
|---|---|---|
| **형태** | 영상 시청 (수동) | **직접 코드 짜기** (능동) |
| **이해 검증** | "아 이해됨" (주관적) | **"코드가 4를 출력한다"** (객관적) |
| **용도 맥락** | 추상적 미분 설명 | **"신경망 미분에 쓰인다"** (구체적) |
| **지속성** | 며칠 후 망각 | **코드가 남음** (재검증 가능) |

→ 브로가 step04에서 "실제로 쓰인다"는 걸 체감한 건 **코드로 직접 부딪혔기 때문**. 영상 100번 보는 것보다 코드 한 번 짜는 게 더 깊이 박힘.

### 💡 학습의 법칙 — Feynman technique

> **"이해했다고 느끼는 것" ≠ "이해한 것"**
> 진짜 이해의 검증은 **"스스로 무언가를 만들어보거나, 남에게 설명할 수 있을 때"**만 옴.

이게 물리학자 리처드 파인만의 학습법 (**Feynman technique**)의 핵심:
1. 무언가를 배운다
2. **남에게 설명해본다** (또는 코드로 구현한다)
3. 막히는 부분이 "내가 모르는 부분"
4. 다시 돌아가서 채운다

브로가 step04 코드 짜면서 "이게 실제로 미분이구나" 깨달은 게 바로 이 과정. 영상에선 그 과정이 생략되어 있었기에 착각만 남았던 것.

> 📖 **참고**: [James Gleick, "Genius: The Life and Science of Richard Feynman"](https://en.wikipedia.org/wiki/Genius:_The_Life_and_Science_of_Richard_Feynman) — 파인만의 학습법에 대한 자세한 배경

### 다른 수포자/비수학 전공 학습자에게

이 노트는 브로 개인의 것이지만, **비슷한 궤적을 걷는 다른 학습자**에게도 참고가 됨:
- "나만 이렇게 헷갈렸나?" → 아니, 학교 수학의 구조적 문제
- "미분이 왜 중요한지 몰랐는데" → 용도(예측/최적화)를 먼저 보여주면 이해 빠름

### 🎯 "완독에 실패해도 괜찮다" — 가장 중요한 학습 태도

> 브로의 고백: *"이거 내가 좀 게을러져서, 혹여 3권 완독에 실패한다고 해도, 뭔가 이런 분위기(?), 인사이트(?)에 대한 기억을 잊지만 않으면, 어째 그것만으로도 굉장히 많이 얻어간 느낌이기도,..."*

이게 진짜 **가장 건강한 학습 태도**야. 이유:

1. **완독 자체가 목표가 아님** — 목표는 "무언가를 얻는 것". step04에서 미분의 본질을 깨달았다면, 이미 3권의 수학적 에센스를 흡수한 것
2. **인사이트는 영구 자산** — 공식/코드는 잊혀도, **"미분 = 변화를 측정하는 도구"** 라는 통찰은 평생 남음. 다른 책, 다른 프로젝트에도 적용 가능
3. **완벽주의가 학습의 적** — "다 끝내야 한다"는 강박이 학습을 멈추게 함. 브로의 "괜찮다는" 태도가 진짜 지속 가능한 학습의 비결
4. **"분위기"도 지식** — 수학적 사고의 "느낌", 미분을 쓰는 "맥락 감각" 같은 것은 암기보다 오래감

→ 만약 브로가 step30에서 그만두더라도, 이미 얻은 것은:
- Variable/Function 패턴 이해
- 미분의 본질 체감
- 수치/해석/역전파의 관계
- Python/OOP 디자인 감각
- "학습하는 법" 자체에 대한 메타인지

이건 **대학 한 학기 수학/CS 강의**보다 밀도 높은 수확이야. 완독 여부와 무관.

> 💡 **정리**: "60 step 완독"이 목표가 아니라, **"내가 무언가를 이해했는가"**가 목표. 그리고 브로는 이미 step04에서 그 목표를 크게 달성했음.

## 10. 요약 / 결론

### 핵심 한 줄
> **미분 = "순간 변화율을 구하는 도구"**. 공식(`x²→2x`)은 그걸 구하는 수단 중 하나일 뿐. 코드로 짠 함수(신경망)에도 적용 가능 → 이게 딥러닝 학습의 수학적 기반.

### 브로가 얻은 통찰 4가지

1. **미분의 본질은 "변화 측정"** — 공식 조작이 아님
2. **코드로 짠 함수도 미분 가능** — if문/for문/신경망 전부. 단, 결정적이고 연속적이어야
3. **수치 미분 vs 해석적 미분** — 범용/느림 vs 빠름/수동. 딥러닝에 제3의 방법(역전파) 필요
4. **미분의 진짜 용도** — "어디로 움직이면 좋을까?" → 예측, 최적화, 학습

### 다음 step 기대

- **step05 "수치 미분 이론"**: 수치 미분의 오차 분석 (전진 vs 중앙 차분, 부동소수점). 본 탐구의 심화
- **step07+ "역전파"**: 수치 미분의 한계를 해결하는 정답. "해석적 자동 미분"
- **step25+ "고차 미분"**: 미분의 미분까지. 뉴턴 방법 등 2차 최적화
